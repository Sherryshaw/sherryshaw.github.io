第一次搭建个人博客遇到的各种坑

搭建一个自己的博客应该是每一个程序猿都想做的事，于是昨天晚上心血来潮！便决定一口气把这个博客搭出来，起初居然还一直以为这是一件简单的事（我一直以为是开一个github账号，然后建一个仓库，设个page页完事），直到开始着手做这个事，才慢慢发现各种坑啊。之后便想着把这个博客搭好后，将遇到的问题分享给大家，如果大家以后遇到类似的问题，希望对大家有帮助。

写在前面

首先，我搭建这个博客用到的是github+hexo的方法，使用了next主题，具体搭建步骤参考了知乎吴润的教程，大家可以直接根据大佬的教程搭建，我在这里仅列举出我在搭建过程中遇到的问题。我使用的系统是linux，所以一些问题可能windows平台是可能没有的。

遇到的问题

安装hexo的npm命令无效。

在安装hexo之前需要安装nodejs和git。
但是在使用npm命令时会发现npm根本没有安装上，这时需要先安装node，node安装成功后npm就可以正常使用了。

npm install -g hexo-cli命令执行卡住。

大多教程里都说这个命令执行后卡一会是正常的，然后昨天在电脑前干等了半个小时他还是不动，于是我就开始找原因。

安装npm install时，长时间停留在fetchMetadata: sill mapToRegistry 处，此处需要在对应的URL处下载一些文件，只要网络稳定，一般花不了多长时间，但是，最近两次操作都是长时间停留，在网上找了一些原因，才发现后面的URL地址可以由淘宝的源代替，原来是资源问题导致的 查看原文

解决方法:
npm config set registry https://registry.npm.taobao.org

这行命令用于将npm的源跟换为taobao的源，然后就可以正常安装了。这里我安装hexo只用了很短的时间。

SSH生成的秘钥与github的秘钥无法匹配

因为保证只有搭建博客的这一台电脑可以向github里进行更新（可能这样安全一些吧），所以需要建立SSH秘钥，本地生成后将文件./ssh/id_rsa.pub的内容复制到github端去（具体操作在那位知乎大佬的教程里有）,然后每次向github部署都会比对你的证书，比对成功方可部署，然而我生成的秘钥却一直比对失败。查找很多资料后发现，其实因为github要比对文件名为github_rsa.pub的秘钥。而我们生成的秘钥文件名为id_rsa.pub，所以会一直失败，改一下名字就好（还有两种方法，但是急于找到解决方法，找不到当时的链接了）。

绑定域名


我是按照如上方式绑定的域名解析，但同时我在配置CNAME文件时出了问题，首先CNAME文件没有后缀！,然后我根据知乎那位大神的方式配置了thzone.cn的CNAME文件（不带WWW的），但是每次部署到github时都会有警告，并且www.thzone.cn 并不能访问成功，所以我又改回了www.thzone.cn 的CNAME文件配置（带WWW的），然后，我成功了。。。

最后

以上就是我遇到的问题，当然只是这些问题相对棘手一些，小问题还是蛮多的，但不至于耽误时间，基本都能查到，这是我第一次搭博客，也是第一次发表博客，总的来说，这个过程还是学到了很多。在之后的时间里我也会积极的去更新。大家有什么疑问也可以给我发邮件，我会及时回复的。我只是一个刚刚开始接触编程的小白，如果我的博客有写的不对不合适的地方，欢迎各位大佬给我反馈意见。^_^
