Leetcode第22题：括号生成

题目

给出 n 代表生成括号的对数，请你写出一个函数，使其能够生成所有可能的并且有效的括号组合。

例如，给出 n = 3，生成结果为：

[
“((()))”,
“(()())”,
“(())()”,
“()(())”,
“()()()”
]

总结

这道题和之前的第20题有效的括号其实很像，不同的点在于只是对相应数量的小括号“()”做全排列，所以该题的思路主要就是结合20题的括号匹配加上递归实现全排列。在实现递归时有以下种情况：

  1. 字符串中将所有括号匹配完后无剩余括号并且匹配的左括号小于3，则只能继续加左括号“（”，因为如果添加右括号“）”则不会有对应的左括号与其匹配。
  2. 字符串中将所有括号匹配完后剩余左括号“（”若干并且匹配的左括号小于3，则可以在此时继续添加左括号或右括号（即分为两只继续进行递归）。
  3. 匹配的左括号等于3，但匹配后剩余的左括号数量仍大于0，则只能继续加右括号“）”，用以匹配剩余未匹配的左括号。
  4. 匹配的左括号等于3，但匹配后剩余的左括号数量等于0，得到其中一个结果，将结果添加到结果集中。

在完成该题的最后阶段，遇到一些问题——无论的怎么改变传入的参数，最后总是会报栈错误，一开始是以为传参的值不对，改了好一会，最后才发现，是因为第六次递归结束后没有及时阻断导致返回至第五次递归时将第二种情况再次递归了一遍，本来应该直接返回至第一次递归的。
