浅析.Net与Java

从8月份开始，我就一直在学习.Net方向的东西，从前端、数据库、语言、框架一条龙。而直到今天，大致的学习内容基本上告一段落，而从我最近完成的case项目来看掌握度也算中规中矩吧。所以今天，我就很想聊一聊这两个方向，也希望我个人的心得能够给大家一些参考。

为什么要学.Net

从市场需求去讲，Java人才溢出，而.Net相对冷门。.Net和Java基本上市场是四六分，而目前的社会现状是：“大街上随便拉一个人都会Java”。所以你只要掌握别人没有掌握的，你就比别人有竞争力。
从技术方面去说，.Net很多地方与java类似，因此学习成本很低，很多地方的思想和Java是互通的，而且比Java更方便，框架的配置很简单，底层直接提供连接DataBase的接口。
其他方面，微软大法好！微软家的东西上手确实快，比如你会用mysql那自然就会了SQL Server，比如你会用Java，那你可以很快速的上手C#，而当你掌握这些时，你会发现他们会组成一个完整的生态链——自家的产品支持起来肯定是最简单方便的。你可以很简单的连接上数据库，你可以很简单的处理office，甚至调用windows一些系统的接口，这些产品链形成的效应，是java所不能比的。

.Net项目涉及到的知识

  1. C#（C SHARP）
    支撑整个项目的面向对象的高级编程语言，官方说是基于c和c++，但是这个语言真的和Java惊人的相似，如果你学习过Java方向，这个语言学习起来会很轻松。

  2. SQL Server
    三大数据库之一，顾名思义用来存储网站的数据。学习他的原因主要还是微软自己的东西，用起来很多情况都很方便，不像其他数据库你可能还需要添加依赖，但使用其他数据库也没有问题。

  3. Html/CSS/Jquery
    web前端三件套，用来搭建网站的前台界面，如果你的项目是web项目，这个是必须要掌握的。尤其是在如今，很多项目都是在网站前端来执行一些过程以减轻服务器的压力，这三项技术变得更加重要

  4. .Net Framework
    .Net说是一个框架，我觉得它更像是对于c#的一个扩展包，提供了很多c#本身并没有的工具类、方法，这让c#从某种程度上更加接近Java。

  5. ADO .Net
    用以在项目中连接数据库，作用和语句的写法类似与Java中的JDBC。

  6. ASP .Net MVC
    web项目的整体框架，MVC就是那个你熟悉的MVC，毕竟都是模型视图控制器，如果你用过SSM框架，你在学习时真的没有压力。

项目流程

  1. 需求分析&概要设计
    因为是老师带着我们，所以本身比较慎重的一步就省略了，老师给我们分发了一份关于研究生管理系统的概要设计。
  2. 数据库设计
    根据概要设计开始设计数据库，这个也不分思路，不管什么数据库，思想基本都一样，在这里不赘述。
  3. 编程阶段
    现在开始步入正题，因为其他步骤也就是软件工程的“V”字模型，而在编程阶段才是每个框架可能有区分的地方。
    • 建立Model类库，也就是模型，存放实体类，其实我个人更喜欢命名为Entity。。。
    • 建立DAL类库，用于连接数据库，首先要在DAL层里创建一个SqlHelper类，用这个类去连接数据库，然后根据实体类一一对应的关系，分别建立DAL类去调用SqlHelper达到连接数据库的目的，其实总结下来，他和JDBC的连接方法很像，可能因为.Net没有Mybatis。
    • 建立BLL类库，BLL是面向工程的最外层，本身调用对应的DAL，自身供外层的Controller调用。
    • 在项目中搭建MVC框架，你在项目新建时可以勾上MVC，这样就不需要进行配置。在项目的文件夹中Controller文件夹存放控制器，View文件夹中存放视图（网页），需要注意的是这里面每个Controller和视图里的文件夹是一一对应的，你在创建控制器时会自动生成。而每个视图文件夹里的视图都必须声明对应控制器里的同名方法，否则会404。

大体的项目流程就是这些，至于其他的本身思想就是和ssm共通的，所以肯定不会感觉到陌生。

最后

三个月的时间，我也不能说是熟练掌握，只是根据自己所学到的来分析分析我认为的.Net，如果想多做业务，可以考虑一下学习这个，但如果想多做研发等等，还是在自己java的基础上多做扩展，多学习些新技术，相比vue、python，和springboot等的新框架，.NET系列还是过气了。
